# Configuration Reference

Complete reference for configuring capitan.

## Creating a Capitan Instance

```go
c := capitan.New(options...)
```

Pass zero or more configuration options to customize behavior.

## Configuration Options

### WithBufferSize

Sets the buffer size for worker queues (per signal).

```go
func WithBufferSize(size int) Option
```

**Parameters**:
- `size` - Number of events to buffer per signal worker

**Default**: 100

**Example**:
```go
// Small buffer (low memory, fast backpressure)
c := capitan.New(capitan.WithBufferSize(10))

// Medium buffer (balanced)
c := capitan.New(capitan.WithBufferSize(100))

// Large buffer (handle traffic spikes)
c := capitan.New(capitan.WithBufferSize(1000))
```

**Guidelines**:

| Buffer Size | Memory | Backpressure | Use Case |
|-------------|--------|--------------|----------|
| 10-50 | Low | Fast | Low-throughput, memory-constrained |
| 100-500 | Medium | Balanced | Most applications |
| 1000+ | High | Slow | High-throughput, traffic spikes |

**Memory Impact**:
```
Per Signal Memory = bufferSize * 200 bytes (approx)

Example:
- 10 signals × 100 buffer × 200 bytes = ~200KB
- 100 signals × 1000 buffer × 200 bytes = ~20MB
```

**When to Adjust**:

**Increase buffer size if**:
- Listeners are slow and you want to queue more events
- You have traffic spikes and want to smooth them
- `Stats().PendingEvents` shows queues often full

**Decrease buffer size if**:
- Memory is constrained
- You want fast backpressure on slow listeners
- You prefer blocking Emit() over unbounded queuing

**Example - Environment-Based**:
```go
func NewCapitan() *capitan.Capitan {
    bufferSize := 100

    if size := os.Getenv("CAPITAN_BUFFER_SIZE"); size != "" {
        if parsed, err := strconv.Atoi(size); err == nil {
            bufferSize = parsed
        }
    }

    return capitan.New(capitan.WithBufferSize(bufferSize))
}
```

### WithSyncMode

Enables synchronous mode where listeners are called directly (no workers).

```go
func WithSyncMode() Option
```

⚠️ **Testing Only**: Use only in tests for deterministic behavior.

**Default**: Async mode (workers + queues)

**Example**:
```go
// Sync mode (testing)
c := capitan.New(capitan.WithSyncMode())

// Async mode (production) - default
c := capitan.New()
```

**Sync Mode Behavior**:
- No worker goroutines
- No event queues
- Listeners called directly in `Emit()` goroutine
- Events processed in registration order
- No event pooling (events not reused)
- `Emit()` blocks until all listeners complete

**When to Use Sync Mode**:

✅ **Use in tests**:
```go
func TestHandler(t *testing.T) {
    c := capitan.New(capitan.WithSyncMode())
    defer c.Shutdown()

    var processed bool
    c.Hook(signal, func(ctx context.Context, e *Event) {
        processed = true
    })

    c.Emit(context.Background(), signal)

    // No need to wait - already processed
    assert.True(t, processed)
}
```

❌ **Don't use in production**:
- Loses async benefits
- Slow listeners block Emit()
- No worker isolation

**Comparison**:

| Feature | Sync Mode | Async Mode |
|---------|-----------|------------|
| **Workers** | No | Yes (per signal) |
| **Queues** | No | Yes (buffered) |
| **Event Pooling** | No | Yes |
| **Ordering** | Sequential | Per-signal sequential |
| **Isolation** | None | Perfect (per signal) |
| **Emit() Latency** | ~500ns | ~1-2µs |
| **Use Case** | Testing | Production |

## Runtime Configuration

### Monitoring Queue Depths

```go
func monitorQueues(c *capitan.Capitan) {
    ticker := time.NewTicker(10 * time.Second)
    defer ticker.Stop()

    for range ticker.C {
        stats := c.Stats()

        for signal, count := range stats.PendingEvents {
            log.Printf("Signal %s: %d pending", signal, count)

            if count > 100 {
                log.Printf("Warning: High queue depth for %s", signal)
            }
        }
    }
}

go monitorQueues(c)
```

### Dynamic Buffer Sizing

Not supported - buffer size is fixed at creation time.

**Workaround**: Create new instance with different size:

```go
oldCapitan := c

// Create new instance with larger buffer
newCapitan := capitan.New(capitan.WithBufferSize(1000))

// Migrate listeners
// ... re-register all hooks

// Shutdown old instance
oldCapitan.Shutdown()

c = newCapitan
```

## Environment Variables

Capitan doesn't read environment variables directly, but you can implement:

```go
package config

import (
    "os"
    "strconv"

    "github.com/zoobzio/capitan"
)

func NewCapitan() *capitan.Capitan {
    var opts []capitan.Option

    // Buffer size from env
    if size := os.Getenv("CAPITAN_BUFFER_SIZE"); size != "" {
        if parsed, err := strconv.Atoi(size); err == nil {
            opts = append(opts, capitan.WithBufferSize(parsed))
        }
    }

    // Sync mode from env (testing)
    if os.Getenv("CAPITAN_SYNC_MODE") == "true" {
        opts = append(opts, capitan.WithSyncMode())
    }

    return capitan.New(opts...)
}
```

Usage:
```bash
# Development
export CAPITAN_BUFFER_SIZE=10

# Production
export CAPITAN_BUFFER_SIZE=500

# Testing
export CAPITAN_SYNC_MODE=true
```

## Configuration Validation

```go
func ValidatedNew(bufferSize int) (*capitan.Capitan, error) {
    if bufferSize < 1 {
        return nil, errors.New("buffer size must be >= 1")
    }

    if bufferSize > 10000 {
        log.Printf("Warning: Very large buffer size (%d)", bufferSize)
    }

    return capitan.New(capitan.WithBufferSize(bufferSize)), nil
}
```

## Defaults

If no options are provided:

```go
c := capitan.New()
```

Defaults are:
- Buffer size: **100** events per signal
- Mode: **Async** (workers + queues)
- Event pooling: **Enabled**

## Best Practices

### 1. Use Default Buffer Size

```go
// ✅ Good: Default is fine for most cases
c := capitan.New()
```

### 2. Tune Based on Metrics

```go
// Monitor queue depths
stats := c.Stats()

for signal, count := range stats.PendingEvents {
    if count > bufferSize*0.8 {
        log.Printf("Consider increasing buffer size for %s", signal)
    }
}
```

### 3. Match Buffer Size to Workload

```go
// Slow listener + high traffic = large buffer
c := capitan.New(capitan.WithBufferSize(1000))

// Fast listener + low traffic = small buffer
c := capitan.New(capitan.WithBufferSize(50))
```

### 4. Only Use Sync Mode in Tests

```go
// ✅ Testing
func TestFeature(t *testing.T) {
    c := capitan.New(capitan.WithSyncMode())
    // ...
}

// ❌ Production
func main() {
    c := capitan.New(capitan.WithSyncMode()) // Don't do this!
}
```

### 5. Single Instance Per Application

```go
// ✅ Good: One instance
var globalCapitan = capitan.New(capitan.WithBufferSize(500))

// ❌ Bad: Multiple instances (confusing)
c1 := capitan.New()
c2 := capitan.New()
```

## Performance Impact

### Buffer Size vs Latency

Small buffer:
- ✅ Low memory
- ✅ Fast backpressure
- ❌ Emit() blocks sooner

Large buffer:
- ❌ High memory
- ❌ Slow backpressure
- ✅ Emit() rarely blocks

### Sync vs Async

Sync mode:
- ✅ Deterministic
- ✅ Lower latency (~500ns)
- ❌ No isolation
- ❌ Blocks Emit()

Async mode:
- ✅ Worker isolation
- ✅ Non-blocking Emit()
- ❌ Slight latency (~1-2µs)
- ❌ Requires Shutdown()

## Troubleshooting

### Queue Fills Up

**Symptom**: `Emit()` blocks frequently

**Solutions**:
1. Increase buffer size
2. Speed up listeners
3. Add more signals (split workload)

### Memory Usage High

**Symptom**: High memory consumption

**Solutions**:
1. Decrease buffer size
2. Reduce number of signals
3. Call Shutdown() to free resources

### Events Processing Slow

**Symptom**: Events take time to process

**Solutions**:
1. Check for slow listeners
2. Increase worker count (split signals)
3. Use async processing in listeners
