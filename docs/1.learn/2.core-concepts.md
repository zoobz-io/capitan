# Core Concepts

## The Event Coordination Model

Capitan organizes event coordination around four core concepts:

1. **Signals** - Named event types
2. **Listeners** - Handlers that respond to specific signals
3. **Observers** - Handlers that watch multiple signals
4. **Workers** - Per-signal goroutines that process events

```
┌──────────────────────────────────────────────────────────┐
│                    Event Flow                            │
└──────────────────────────────────────────────────────────┘

Application Code
      │
      ▼
  c.Emit(ctx, signal, fields...)
      │
      ▼
┌─────────────┐
│ Event Pool  │ (sync.Pool - zero allocation)
└─────────────┘
      │
      ▼
┌─────────────────────────────────────────────────────────┐
│                  Signal Router                          │
│  (Dispatches to signal-specific worker + observers)    │
└─────────────────────────────────────────────────────────┘
      │
      ├───────────────┬────────────────┐
      ▼               ▼                ▼
┌──────────┐    ┌──────────┐    ┌──────────┐
│ Worker 1 │    │ Worker 2 │    │ Observer │
│  Queue   │    │  Queue   │    │  Direct  │
└──────────┘    └──────────┘    └──────────┘
      │               │                │
      ▼               ▼                ▼
  Listeners       Listeners        Handler
```

## Signals

A **Signal** is a named event type that represents something that happened in your system.

### Creating Signals

```go
// Create signal with name and description
orderPlaced := capitan.NewSignal("order.placed", "Customer placed an order")
userRegistered := capitan.NewSignal("user.registered", "New user registered")
paymentFailed := capitan.NewSignal("payment.failed", "Payment processing failed")
```

### Signal Identity

Signals are identified by their name string. Two signals with the same name are considered identical:

```go
sig1 := capitan.NewSignal("order.placed", "Description 1")
sig2 := capitan.NewSignal("order.placed", "Description 2")

// sig1 == sig2 (same name)
```

**Best Practice**: Define signals as package-level constants:

```go
package events

var (
    OrderPlaced    = capitan.NewSignal("order.placed", "Order placed")
    OrderShipped   = capitan.NewSignal("order.shipped", "Order shipped")
    OrderDelivered = capitan.NewSignal("order.delivered", "Order delivered")
)
```

### Signal Naming Conventions

Use hierarchical naming with dots:

```go
// Good: Hierarchical, specific
"order.placed"
"order.payment.processed"
"user.profile.updated"
"inventory.stock.low"

// Avoid: Generic, unclear
"event"
"update"
"process"
```

## Events

An **Event** is an instance of a signal with associated data fields and metadata.

### Event Structure

```go
type Event struct {
    // Signal that triggered this event
    Signal() Signal

    // When the event was created
    Timestamp() time.Time

    // Optional severity level
    Severity() Severity

    // Typed data fields
    Fields() []Field
}
```

### Emitting Events

```go
// Simple emission
c.Emit(ctx, orderPlaced)

// With typed fields
userKey := capitan.NewStringKey("user_id")
amountKey := capitan.NewIntKey("amount")

c.Emit(ctx, orderPlaced,
    userKey.Field("user-123"),
    amountKey.Field(9999),
)

// With severity
c.EmitWithSeverity(ctx, capitan.SeverityError, paymentFailed,
    userKey.Field("user-123"),
)
```

### Event Pooling

Events are pooled via `sync.Pool` for performance:

```go
// Events are automatically:
// 1. Retrieved from pool
// 2. Populated with data
// 3. Sent to workers
// 4. Returned to pool after processing

// No allocations in hot paths!
```

**Important**: Events are pooled and reused after processing. Never store event pointers - copy the data you need. See [Architecture - Defensive Copying](./3.architecture.md#defensive-copying) for details.

## Fields

**Fields** are strongly-typed key-value pairs attached to events.

### Field Types

```go
// String fields
nameKey := capitan.NewStringKey("name")
c.Emit(ctx, signal, nameKey.Field("John"))

// Integer fields
ageKey := capitan.NewIntKey("age")
c.Emit(ctx, signal, ageKey.Field(25))

// Float fields
priceKey := capitan.NewFloatKey("price")
c.Emit(ctx, signal, priceKey.Field(99.99))

// Boolean fields
activeKey := capitan.NewBoolKey("active")
c.Emit(ctx, signal, activeKey.Field(true))

// Duration fields
timeoutKey := capitan.NewDurationKey("timeout")
c.Emit(ctx, signal, timeoutKey.Field(5*time.Second))

// Time fields
createdKey := capitan.NewTimeKey("created_at")
c.Emit(ctx, signal, createdKey.Field(time.Now()))

// Any type (loses type safety)
dataKey := capitan.NewAnyKey("metadata")
c.Emit(ctx, signal, dataKey.Field(map[string]string{"key": "value"}))
```

### Extracting Fields

```go
userKey := capitan.NewStringKey("user_id")
amountKey := capitan.NewIntKey("amount")

c.Hook(orderPlaced, func(ctx context.Context, e *capitan.Event) {
    // Extract with type safety
    userID := userKey.Extract(e)   // string
    amount := amountKey.Extract(e) // int

    processOrder(userID, amount)
})
```

### Field Best Practices

1. **Define keys as package constants**:
```go
package fields

var (
    UserID    = capitan.NewStringKey("user_id")
    OrderID   = capitan.NewStringKey("order_id")
    Amount    = capitan.NewIntKey("amount")
    Timestamp = capitan.NewTimeKey("timestamp")
)
```

2. **Use specific types over `any`**:
```go
// ✅ Good: Type-safe
amountKey := capitan.NewIntKey("amount")

// ❌ Avoid: Loses type safety
amountKey := capitan.NewAnyKey("amount")
```

3. **Keep field names consistent**:
```go
// ✅ Good: Consistent naming
"user_id", "order_id", "product_id"

// ❌ Avoid: Inconsistent
"userId", "order_id", "ProductID"
```

## Listeners

A **Listener** is a handler function that responds to a specific signal.

### Creating Listeners

```go
// Hook a listener to a signal
listener := c.Hook(orderPlaced, func(ctx context.Context, e *capitan.Event) {
    log.Printf("Order placed: %v", e.Fields())
})

// Multiple listeners per signal
c.Hook(orderPlaced, sendConfirmationEmail)
c.Hook(orderPlaced, updateInventory)
c.Hook(orderPlaced, notifyWarehouse)
```

### Listener Guarantees

1. **Ordered Execution**: Listeners for the same signal execute in registration order
2. **Isolation**: Each signal has its own worker - no cross-signal blocking
3. **Context Propagation**: Context from `Emit()` flows through to handlers

```go
c.Hook(signal1, slowHandler)  // Doesn't block signal2
c.Hook(signal2, fastHandler)  // Runs independently
```

### Listener Lifecycle

```go
// Create listener
listener := c.Hook(signal, handler)

// Listener processes events
c.Emit(ctx, signal, fields...)

// Close when done
listener.Close()

// Events emitted after Close() won't reach this listener
```

### Closing Listeners

```go
listener := c.Hook(signal, handler)

// Close listener when no longer needed
listener.Close()

// Safe to call multiple times
listener.Close()
listener.Close()
```

## Observers

An **Observer** watches multiple signals without pre-registration.

### Creating Observers

```go
// Observe ALL signals
observer := c.Observe(func(ctx context.Context, e *capitan.Event) {
    log.Printf("[AUDIT] %s at %s", e.Signal().Name(), e.Timestamp())
})

// Observe specific signals (whitelist)
observer := c.Observe(auditHandler,
    orderPlaced,
    paymentProcessed,
    userRegistered,
)
```

### Observer vs Listener

| Feature | Listener | Observer |
|---------|----------|----------|
| **Target** | Single signal | All signals (or whitelist) |
| **Registration** | Explicit per signal | Dynamic |
| **Use Case** | Business logic | Cross-cutting concerns |
| **Worker** | Signal's worker | Direct invocation |
| **Order** | Guaranteed | No ordering guarantee |

### Observer Use Cases

**Audit Logging**
```go
observer := c.Observe(func(ctx context.Context, e *capitan.Event) {
    auditLog.Record(AuditEntry{
        Signal:    e.Signal().Name(),
        Timestamp: e.Timestamp(),
        Fields:    e.Fields(),
        Severity:  e.Severity(),
    })
})
```

**Metrics Collection**
```go
observer := c.Observe(func(ctx context.Context, e *capitan.Event) {
    metrics.Inc("events_total", map[string]string{
        "signal": e.Signal().Name(),
    })
})
```

**Development Debugging**
```go
if os.Getenv("DEBUG_EVENTS") == "true" {
    observer := c.Observe(func(ctx context.Context, e *capitan.Event) {
        fmt.Printf("[DEBUG] %s: %+v\n", e.Signal().Name(), e.Fields())
    })
    defer observer.Close()
}
```

### Observer Whitelisting

```go
// Only observe sensitive operations
sensitiveObserver := c.Observe(
    func(ctx context.Context, e *capitan.Event) {
        securityLog.Record(e)
    },
    userLogin,
    userLogout,
    passwordChanged,
    permissionChanged,
)
```

## Workers

Each signal has a dedicated **Worker** goroutine that processes events sequentially.

### Worker Architecture

```
Signal A               Signal B               Signal C
┌────────────┐        ┌────────────┐        ┌────────────┐
│  Worker 1  │        │  Worker 2  │        │  Worker 3  │
│            │        │            │        │            │
│ ┌────────┐ │        │ ┌────────┐ │        │ ┌────────┐ │
│ │ Queue  │ │        │ │ Queue  │ │        │ │ Queue  │ │
│ │ [E][E] │ │        │ │ [E][E] │ │        │ │ [E]    │ │
│ └────────┘ │        │ └────────┘ │        │ └────────┘ │
│     ▼      │        │     ▼      │        │     ▼      │
│ Listener 1 │        │ Listener 1 │        │ Listener 1 │
│ Listener 2 │        │ Listener 2 │        │ Listener 2 │
│ Listener 3 │        │            │        │            │
└────────────┘        └────────────┘        └────────────┘

Independent execution - no contention
```

### Worker Guarantees

1. **Sequential Processing**: Events processed in order per signal
2. **Isolation**: One signal's slow handler doesn't affect others
3. **Buffering**: Configurable queue size per worker
4. **Clean Shutdown**: `Shutdown()` drains queues before exiting

### Worker Creation

Workers are created lazily:

```go
// No worker yet
sig := capitan.NewSignal("test", "Test signal")

// Worker created on first Hook() or Emit()
c.Hook(sig, handler) // Creates worker

// Or
c.Emit(ctx, sig, fields...) // Creates worker if needed
```

### Backpressure

When a worker's queue fills up:

```go
c := capitan.New(capitan.WithBufferSize(10)) // 10-event buffer per worker

// If queue fills, Emit() blocks until space available
c.Emit(ctx, signal, fields...) // May block if queue full
```

### Worker Shutdown

```go
// Shutdown waits for all workers to drain queues
c.Shutdown()

// All queued events are processed
// All listeners complete
// Resources are cleaned up
```

## Context Propagation

Context flows from `Emit()` through workers to handlers:

```go
// Create context with values
ctx := context.WithValue(context.Background(), "trace_id", "abc-123")

// Emit with context
c.Emit(ctx, signal, fields...)

// Handler receives same context
c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    traceID := ctx.Value("trace_id").(string) // "abc-123"

    // Context cancellation is respected
    if ctx.Err() != nil {
        return // Context cancelled
    }
})
```

### Context Cancellation

```go
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

c.Emit(ctx, signal, fields...)

c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    // Check for cancellation
    select {
    case <-ctx.Done():
        return // Exit early
    default:
        // Process event
    }
})
```

## Severity Levels

Events can have severity for filtering and prioritization:

```go
const (
    SeverityDebug Severity = iota
    SeverityInfo
    SeverityWarn
    SeverityError
    SeverityCritical
)

// Emit with severity
c.EmitWithSeverity(ctx, capitan.SeverityError, paymentFailed, fields...)

// Filter by severity
c.Hook(errorSignal, func(ctx context.Context, e *capitan.Event) {
    if e.Severity() >= capitan.SeverityError {
        alertOncall(e)
    }
})
```

## Best Practices

### 1. Signal Organization

Group related signals:

```go
package events

// Order lifecycle
var (
    OrderPlaced    = capitan.NewSignal("order.placed", "Order placed")
    OrderValidated = capitan.NewSignal("order.validated", "Order validated")
    OrderShipped   = capitan.NewSignal("order.shipped", "Order shipped")
    OrderDelivered = capitan.NewSignal("order.delivered", "Order delivered")
)

// User lifecycle
var (
    UserRegistered = capitan.NewSignal("user.registered", "User registered")
    UserVerified   = capitan.NewSignal("user.verified", "User verified")
    UserDeactivated = capitan.NewSignal("user.deactivated", "User deactivated")
)
```

### 2. Field Key Reuse

Define field keys once:

```go
package fields

var (
    UserID    = capitan.NewStringKey("user_id")
    OrderID   = capitan.NewStringKey("order_id")
    Amount    = capitan.NewIntKey("amount")
    Timestamp = capitan.NewTimeKey("timestamp")
)

// Use across signals
c.Emit(ctx, orderPlaced, fields.UserID.Field("u1"), fields.Amount.Field(100))
c.Emit(ctx, userRegistered, fields.UserID.Field("u1"), fields.Timestamp.Field(time.Now()))
```

### 3. Listener Cleanup

Close listeners when components shut down:

```go
type Service struct {
    listeners []*capitan.Listener
}

func (s *Service) Start(c *capitan.Capitan) {
    s.listeners = append(s.listeners,
        c.Hook(signal1, s.handler1),
        c.Hook(signal2, s.handler2),
    )
}

func (s *Service) Stop() {
    for _, l := range s.listeners {
        l.Close()
    }
}
```

### 4. Observer Lifecycle

Close observers when done:

```go
observer := c.Observe(debugHandler)
defer observer.Close()

// Run tests or debug session
runTests()
```

### 5. Error Handling

Handle errors in listeners gracefully:

```go
c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    if err := processEvent(e); err != nil {
        log.Printf("Error processing event: %v", err)

        // Optionally emit error signal
        c.Emit(ctx, processingFailed,
            errorKey.Field(err.Error()),
            signalKey.Field(e.Signal().Name()),
        )
    }
})
```
