# Observability

*Recipe: Build centralized logging and metrics using capitan observers.*

Capitan's observer pattern enables centralized instrumentation without coupling your components together.

## The Scenario

Consider an e-commerce service with three components:

- **Orders** - Creates and manages orders
- **Payments** - Processes payments
- **Inventory** - Tracks stock levels

Each component operates independently. None should know about logging infrastructure, metrics collection, or tracing systems.

## Instrumenting Components

First, each component defines its own signals and emits events during normal operation:

```go
// orders/signals.go
package orders

import "github.com/zoobzio/capitan"

var (
    OrderCreated   = capitan.NewSignal("order.created", "New order placed")
    OrderConfirmed = capitan.NewSignal("order.confirmed", "Order confirmed after payment")
    OrderCanceled  = capitan.NewSignal("order.canceled", "Order was canceled")
)

var (
    OrderID    = capitan.NewStringKey("order_id")
    CustomerID = capitan.NewStringKey("customer_id")
    Total      = capitan.NewFloat64Key("total")
)
```

```go
// orders/service.go
package orders

import (
    "context"
    "github.com/zoobzio/capitan"
)

func CreateOrder(ctx context.Context, customerID string, total float64) string {
    orderID := generateOrderID()

    // Business logic...

    capitan.Emit(ctx, OrderCreated,
        OrderID.Field(orderID),
        CustomerID.Field(customerID),
        Total.Field(total),
    )

    return orderID
}
```

```go
// payments/signals.go
package payments

import "github.com/zoobzio/capitan"

var (
    PaymentProcessed = capitan.NewSignal("payment.processed", "Payment successfully processed")
    PaymentFailed    = capitan.NewSignal("payment.failed", "Payment processing failed")
)

var (
    PaymentID = capitan.NewStringKey("payment_id")
    OrderID   = capitan.NewStringKey("order_id")
    Amount    = capitan.NewFloat64Key("amount")
    ErrReason = capitan.NewStringKey("error_reason")
)
```

```go
// inventory/signals.go
package inventory

import "github.com/zoobzio/capitan"

var (
    StockReserved = capitan.NewSignal("inventory.reserved", "Stock reserved for order")
    StockReleased = capitan.NewSignal("inventory.released", "Reserved stock released")
    LowStock      = capitan.NewSignal("inventory.low_stock", "Stock level below threshold")
)

var (
    ProductID = capitan.NewStringKey("product_id")
    Quantity  = capitan.NewIntKey("quantity")
    Remaining = capitan.NewIntKey("remaining")
)
```

Each package emits events independently. No package imports another's signals. No shared logging dependency.

## The Observer

At application startup, attach a single observer that captures everything:

```go
// main.go
package main

import (
    "context"
    "encoding/json"
    "log"
    "time"

    "github.com/zoobzio/capitan"
)

func main() {
    // Attach observer before any events are emitted
    capitan.Observe(func(ctx context.Context, e *capitan.Event) {
        entry := map[string]any{
            "timestamp": e.Timestamp().Format(time.RFC3339),
            "signal":    e.Signal().Name(),
            "severity":  e.Severity(),
            "message":   e.Signal().Description(),
            "fields":    extractFields(e),
        }

        data, _ := json.Marshal(entry)
        log.Println(string(data))
    })

    // Start your application...

    capitan.Shutdown()
}

func extractFields(e *capitan.Event) map[string]any {
    result := make(map[string]any)
    for _, f := range e.Fields() {
        result[f.Key().Name()] = f.Value()
    }
    return result
}
```

## What This Produces

When the application runs, every event flows through the observer:

```json
{"timestamp":"2024-01-15T10:30:00Z","signal":"order.created","severity":"INFO","message":"New order placed","fields":{"order_id":"ORD-123","customer_id":"CUST-456","total":99.99}}
{"timestamp":"2024-01-15T10:30:01Z","signal":"inventory.reserved","severity":"INFO","message":"Stock reserved for order","fields":{"product_id":"PROD-789","quantity":2,"remaining":48}}
{"timestamp":"2024-01-15T10:30:02Z","signal":"payment.processed","severity":"INFO","message":"Payment successfully processed","fields":{"payment_id":"PAY-001","order_id":"ORD-123","amount":99.99}}
{"timestamp":"2024-01-15T10:30:02Z","signal":"order.confirmed","severity":"INFO","message":"Order confirmed after payment","fields":{"order_id":"ORD-123"}}
```

Structured logs from every component, zero coupling between them.

## Extending the Pattern

The observer receives every event. Route them anywhere:

```go
// Send to multiple destinations
capitan.Observe(func(ctx context.Context, e *capitan.Event) {
    // Structured logging
    logger.Info(e.Signal().Description(),
        "signal", e.Signal().Name(),
        "fields", extractFields(e))

    // Metrics
    metrics.Increment("events_total", "signal", e.Signal().Name())

    // Tracing
    if span := trace.SpanFromContext(ctx); span != nil {
        span.AddEvent(e.Signal().Name())
    }
})
```

The components remain unchanged. Add or remove instrumentation at the edges, not in the business logic.

## Filtering by Signal

Observe specific signals when you don't need everything:

```go
// Only observe payment events
capitan.Observe(func(ctx context.Context, e *capitan.Event) {
    alertOnPaymentFailure(e)
}, payments.PaymentProcessed, payments.PaymentFailed)
```

The observer receives only events from the whitelisted signals.
