# Workflows

*Recipe: Build event-driven choreography where listeners emit events to coordinate multi-step processes.*

Listeners can emit events. This creates workflows where one signal triggers the next, coordinating multiple components without direct coupling.

## The Scenario

Building on the e-commerce service from [Observability](1.observability.md), we now want components to react to each other:

1. Order created → Reserve inventory
2. Payment processed → Confirm order, send notification
3. Payment failed → Release inventory, notify customer
4. Low stock → Alert operations team

Each component listens and responds, but none imports another directly.

## Defining the Signals

Each component owns its signals:

```go
// orders/signals.go
package orders

import "github.com/zoobzio/capitan"

var (
    OrderCreated   = capitan.NewSignal("order.created", "New order placed")
    OrderConfirmed = capitan.NewSignal("order.confirmed", "Order confirmed after payment")
    OrderCanceled  = capitan.NewSignal("order.canceled", "Order was canceled")
)

var (
    OrderID    = capitan.NewStringKey("order_id")
    CustomerID = capitan.NewStringKey("customer_id")
    ProductID  = capitan.NewStringKey("product_id")
    Quantity   = capitan.NewIntKey("quantity")
    Total      = capitan.NewFloat64Key("total")
)
```

## Wiring the Workflow

At application startup, hook listeners that bridge components:

```go
// main.go
package main

import (
    "context"
    "github.com/zoobzio/capitan"

    "myapp/inventory"
    "myapp/notifications"
    "myapp/orders"
    "myapp/payments"
)

func main() {
    // Order created → Reserve inventory
    capitan.Hook(orders.OrderCreated, func(ctx context.Context, e *capitan.Event) {
        productID, _ := orders.ProductID.From(e)
        orderID, _ := orders.OrderID.From(e)
        quantity, _ := orders.Quantity.From(e)

        remaining := reserveStock(productID, quantity)

        capitan.Emit(ctx, inventory.StockReserved,
            inventory.ProductID.Field(productID),
            inventory.OrderID.Field(orderID),
            inventory.Quantity.Field(quantity),
            inventory.Remaining.Field(remaining),
        )

        // Check for low stock condition
        if remaining < 10 {
            capitan.Warn(ctx, inventory.LowStock,
                inventory.ProductID.Field(productID),
                inventory.Remaining.Field(remaining),
            )
        }
    })

    // Payment processed → Confirm order, notify customer
    capitan.Hook(payments.PaymentProcessed, func(ctx context.Context, e *capitan.Event) {
        orderID, _ := payments.OrderID.From(e)

        order := confirmOrder(orderID)

        capitan.Emit(ctx, orders.OrderConfirmed,
            orders.OrderID.Field(orderID),
        )

        sendNotification(order.CustomerID, "email", "order_confirmed")

        capitan.Emit(ctx, notifications.NotificationSent,
            notifications.CustomerID.Field(order.CustomerID),
            notifications.Channel.Field("email"),
            notifications.Template.Field("order_confirmed"),
        )
    })

    // Payment failed → Release inventory, notify customer
    capitan.Hook(payments.PaymentFailed, func(ctx context.Context, e *capitan.Event) {
        orderID, _ := payments.OrderID.From(e)
        reason, _ := payments.Reason.From(e)

        order := getOrder(orderID)
        releaseStock(order.ProductID, order.Quantity)

        capitan.Emit(ctx, inventory.StockReleased,
            inventory.ProductID.Field(order.ProductID),
            inventory.OrderID.Field(orderID),
            inventory.Quantity.Field(order.Quantity),
        )

        capitan.Emit(ctx, orders.OrderCanceled,
            orders.OrderID.Field(orderID),
        )

        sendNotification(order.CustomerID, "email", "payment_failed")

        capitan.Emit(ctx, notifications.NotificationSent,
            notifications.CustomerID.Field(order.CustomerID),
            notifications.Channel.Field("email"),
            notifications.Template.Field("payment_failed"),
        )
    })

    // Low stock → Alert operations
    capitan.Hook(inventory.LowStock, func(ctx context.Context, e *capitan.Event) {
        productID, _ := inventory.ProductID.From(e)
        remaining, _ := inventory.Remaining.From(e)

        alertOps(productID, remaining)
    })

    // Start application...

    capitan.Shutdown()
}
```

## The Flow

A successful order triggers this sequence:

```
order.created
    └→ inventory.reserved
        └→ inventory.low_stock (if threshold breached)

payment.processed
    └→ order.confirmed
    └→ notification.sent
```

A failed payment triggers:

```
payment.failed
    └→ inventory.released
    └→ order.canceled
    └→ notification.sent
```

Each signal is independent. Each listener reacts to one signal and emits others. The workflow emerges from composition.

## Characteristics

**Decoupled** - The orders package doesn't import inventory or notifications. Signal definitions are the only shared contract.

**Traceable** - Every step emits a signal. Combined with an observer, you get a complete audit trail of the workflow.

**Extensible** - Add a new listener without modifying existing code. Hook `order.confirmed` to trigger loyalty points, analytics, or fraud detection.

**Ordered** - Events to the same signal process in order. A second order won't reserve stock until the first reservation completes.

## Error Handling

Listeners can emit error-severity events to signal failures:

```go
capitan.Hook(orders.OrderCreated, func(ctx context.Context, e *capitan.Event) {
    productID, _ := orders.ProductID.From(e)
    orderID, _ := orders.OrderID.From(e)
    quantity, _ := orders.Quantity.From(e)

    remaining, err := tryReserveStock(productID, quantity)
    if err != nil {
        capitan.Error(ctx, inventory.ReservationFailed,
            inventory.ProductID.Field(productID),
            inventory.OrderID.Field(orderID),
            inventory.ErrReason.Field(err.Error()),
        )
        return
    }

    capitan.Emit(ctx, inventory.StockReserved,
        inventory.ProductID.Field(productID),
        inventory.OrderID.Field(orderID),
        inventory.Quantity.Field(quantity),
        inventory.Remaining.Field(remaining),
    )
})
```

Another listener handles the failure:

```go
capitan.Hook(inventory.ReservationFailed, func(ctx context.Context, e *capitan.Event) {
    orderID, _ := inventory.OrderID.From(e)
    reason, _ := inventory.ErrReason.From(e)

    cancelOrder(orderID, reason)
    notifyCustomerOutOfStock(orderID)
})
```

The error path is just another workflow branch.
