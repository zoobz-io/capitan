# Distribution

> For a message broker implementation of this pattern, see [herald](https://github.com/zoobzio/herald).

*Recipe: Build a broker that bridges capitan to external message systems like Kafka or NATS.*

Capitan coordinates events within a process. A broker pattern extends this to distributed systems—consuming external events, processing them locally, and publishing results back.

## The Pattern

```
┌─────────────────┐     ┌─────────────────┐     ┌─────────────────┐
│  External       │     │    Capitan      │     │  External       │
│  Message Bus    │────▶│    Workflows    │────▶│  Message Bus    │
│  (Kafka, NATS)  │     │                 │     │  (Kafka, NATS)  │
└─────────────────┘     └─────────────────┘     └─────────────────┘
        │                       │                       ▲
        │                       │                       │
   order.created          local processing        order.completed
   (inbound)              with capitan            (outbound)
```

The broker binds external topics to capitan signals. Inbound messages become local events. Outbound signals publish to external topics.

## The Scenario

Continuing the e-commerce example from [Workflows](2.workflows.md), orders now arrive from a Kafka topic managed by another team. After processing, we publish completion events back to Kafka for downstream consumers.

## Defining the Broker

A broker translates between external messages and capitan signals:

```go
// broker/broker.go
package broker

import (
    "context"
    "encoding/json"

    "github.com/zoobzio/capitan"
)

// Binding maps an external topic to a capitan signal with a decoder.
type Binding struct {
    Topic  string
    Signal capitan.Signal
    Decode func([]byte) ([]capitan.Field, error)
}

// Outbound maps a capitan signal to an external topic with an encoder.
type Outbound struct {
    Signal capitan.Signal
    Topic  string
    Encode func(*capitan.Event) ([]byte, error)
}

// Broker bridges capitan to an external message system.
type Broker struct {
    consumer  Consumer
    producer  Producer
    bindings  []Binding
    outbounds []Outbound
}

// Consumer represents any message consumer (Kafka, NATS, etc.)
type Consumer interface {
    Subscribe(topic string, handler func([]byte)) error
    Close() error
}

// Producer represents any message producer.
type Producer interface {
    Publish(topic string, data []byte) error
    Close() error
}

func New(consumer Consumer, producer Producer) *Broker {
    return &Broker{
        consumer: consumer,
        producer: producer,
    }
}

// Bind maps an external topic to a capitan signal.
func (b *Broker) Bind(binding Binding) {
    b.bindings = append(b.bindings, binding)
}

// Publish maps a capitan signal to an external topic.
func (b *Broker) Publish(outbound Outbound) {
    b.outbounds = append(b.outbounds, outbound)
}

// Start begins consuming and publishing.
func (b *Broker) Start(ctx context.Context) error {
    // Subscribe to inbound topics
    for _, binding := range b.bindings {
        bind := binding // capture for closure
        err := b.consumer.Subscribe(bind.Topic, func(data []byte) {
            fields, err := bind.Decode(data)
            if err != nil {
                return // log error in production
            }
            capitan.Emit(ctx, bind.Signal, fields...)
        })
        if err != nil {
            return err
        }
    }

    // Hook outbound signals
    for _, outbound := range b.outbounds {
        out := outbound // capture for closure
        capitan.Hook(out.Signal, func(ctx context.Context, e *capitan.Event) {
            data, err := out.Encode(e)
            if err != nil {
                return // log error in production
            }
            b.producer.Publish(out.Topic, data)
        })
    }

    return nil
}

func (b *Broker) Close() error {
    b.consumer.Close()
    b.producer.Close()
    return nil
}
```

## Wiring the Application

Bind external Kafka topics to capitan signals:

```go
// main.go
package main

import (
    "context"
    "encoding/json"

    "github.com/zoobzio/capitan"

    "myapp/broker"
    "myapp/inventory"
    "myapp/notifications"
    "myapp/orders"
    "myapp/payments"
)

func main() {
    ctx := context.Background()

    // Create Kafka consumer/producer (implementation details omitted)
    consumer := newKafkaConsumer("localhost:9092", "order-processor")
    producer := newKafkaProducer("localhost:9092")

    // Create broker
    b := broker.New(consumer, producer)

    // Bind inbound: Kafka topic → capitan signal
    b.Bind(broker.Binding{
        Topic:  "orders.created",
        Signal: orders.OrderCreated,
        Decode: decodeOrderCreated,
    })

    b.Bind(broker.Binding{
        Topic:  "payments.result",
        Signal: payments.PaymentProcessed, // or PaymentFailed based on payload
        Decode: decodePaymentResult,
    })

    // Bind outbound: capitan signal → Kafka topic
    b.Publish(broker.Outbound{
        Signal: orders.OrderConfirmed,
        Topic:  "orders.confirmed",
        Encode: encodeOrderConfirmed,
    })

    b.Publish(broker.Outbound{
        Signal: orders.OrderCanceled,
        Topic:  "orders.canceled",
        Encode: encodeOrderCanceled,
    })

    // Wire local workflows (same as before)
    capitan.Hook(orders.OrderCreated, func(ctx context.Context, e *capitan.Event) {
        productID, _ := orders.ProductID.From(e)
        orderID, _ := orders.OrderID.From(e)
        quantity, _ := orders.Quantity.From(e)

        remaining := reserveStock(productID, quantity)

        capitan.Emit(ctx, inventory.StockReserved,
            inventory.ProductID.Field(productID),
            inventory.OrderID.Field(orderID),
            inventory.Quantity.Field(quantity),
            inventory.Remaining.Field(remaining),
        )
    })

    capitan.Hook(payments.PaymentProcessed, func(ctx context.Context, e *capitan.Event) {
        orderID, _ := payments.OrderID.From(e)
        confirmOrder(orderID)

        capitan.Emit(ctx, orders.OrderConfirmed,
            orders.OrderID.Field(orderID),
        )
    })

    capitan.Hook(payments.PaymentFailed, func(ctx context.Context, e *capitan.Event) {
        orderID, _ := payments.OrderID.From(e)
        order := getOrder(orderID)

        releaseStock(order.ProductID, order.Quantity)

        capitan.Emit(ctx, orders.OrderCanceled,
            orders.OrderID.Field(orderID),
        )
    })

    // Start broker
    b.Start(ctx)

    // Run until interrupted...

    b.Close()
    capitan.Shutdown()
}

// Decoders translate external JSON to capitan fields
func decodeOrderCreated(data []byte) ([]capitan.Field, error) {
    var msg struct {
        OrderID    string  `json:"order_id"`
        CustomerID string  `json:"customer_id"`
        ProductID  string  `json:"product_id"`
        Quantity   int     `json:"quantity"`
        Total      float64 `json:"total"`
    }
    if err := json.Unmarshal(data, &msg); err != nil {
        return nil, err
    }
    return []capitan.Field{
        orders.OrderID.Field(msg.OrderID),
        orders.CustomerID.Field(msg.CustomerID),
        orders.ProductID.Field(msg.ProductID),
        orders.Quantity.Field(msg.Quantity),
        orders.Total.Field(msg.Total),
    }, nil
}

func decodePaymentResult(data []byte) ([]capitan.Field, error) {
    var msg struct {
        PaymentID string  `json:"payment_id"`
        OrderID   string  `json:"order_id"`
        Amount    float64 `json:"amount"`
        Success   bool    `json:"success"`
        Reason    string  `json:"reason,omitempty"`
    }
    if err := json.Unmarshal(data, &msg); err != nil {
        return nil, err
    }
    return []capitan.Field{
        payments.PaymentID.Field(msg.PaymentID),
        payments.OrderID.Field(msg.OrderID),
        payments.Amount.Field(msg.Amount),
        payments.Reason.Field(msg.Reason),
    }, nil
}

// Encoders translate capitan events to external JSON
func encodeOrderConfirmed(e *capitan.Event) ([]byte, error) {
    orderID, _ := orders.OrderID.From(e)
    return json.Marshal(map[string]any{
        "order_id":  orderID,
        "status":    "confirmed",
        "timestamp": e.Timestamp(),
    })
}

func encodeOrderCanceled(e *capitan.Event) ([]byte, error) {
    orderID, _ := orders.OrderID.From(e)
    return json.Marshal(map[string]any{
        "order_id":  orderID,
        "status":    "canceled",
        "timestamp": e.Timestamp(),
    })
}
```

## The Flow

```
Kafka: orders.created
    │
    ▼
capitan: order.created
    │
    ├──▶ inventory.reserved (local)
    │
    ▼
Kafka: payments.result
    │
    ▼
capitan: payment.processed (or payment.failed)
    │
    ├──▶ order.confirmed ───▶ Kafka: orders.confirmed
    │
    └──▶ order.canceled ────▶ Kafka: orders.canceled
```

External events enter through the broker, flow through local capitan workflows, and results publish back to external topics.

## Characteristics

**Boundary isolation** - The broker is the only code that knows about Kafka. Workflows remain pure capitan—testable without infrastructure.

**Bidirectional** - Consume and produce through the same abstraction. The broker handles both directions.

**Adaptable** - Swap Kafka for NATS, RabbitMQ, or Redis Streams by implementing the Consumer/Producer interfaces. Workflows don't change.

**Type-safe internally** - External JSON maps to typed fields at the boundary. Inside capitan, everything is typed.

## Testing Without Infrastructure

Mock the broker interfaces for unit tests:

```go
type mockConsumer struct {
    handlers map[string]func([]byte)
}

func (m *mockConsumer) Subscribe(topic string, handler func([]byte)) error {
    m.handlers[topic] = handler
    return nil
}

func (m *mockConsumer) Simulate(topic string, data []byte) {
    if h, ok := m.handlers[topic]; ok {
        h(data)
    }
}

func (m *mockConsumer) Close() error { return nil }

type mockProducer struct {
    published []struct {
        Topic string
        Data  []byte
    }
}

func (m *mockProducer) Publish(topic string, data []byte) error {
    m.published = append(m.published, struct {
        Topic string
        Data  []byte
    }{topic, data})
    return nil
}

func (m *mockProducer) Close() error { return nil }
```

Inject mocks, simulate inbound messages, assert outbound publications. The workflow logic tests without Kafka running.
