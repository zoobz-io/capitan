# Concurrency Guide

Understanding thread safety and concurrent patterns in capitan.

## Core Guarantees

Capitan provides strong concurrency guarantees:

1. **Thread-Safe API**: All public methods are safe to call concurrently
2. **Per-Signal Workers**: Each signal has an independent worker goroutine
3. **Ordered Delivery**: Events for the same signal are processed in order
4. **Worker Isolation**: One signal's slow handler doesn't block others

## Per-Signal Workers

Each signal gets its own worker goroutine:

```
Signal A          Signal B          Signal C
┌────────┐       ┌────────┐       ┌────────┐
│Worker 1│       │Worker 2│       │Worker 3│
│ Queue  │       │ Queue  │       │ Queue  │
└────────┘       └────────┘       └────────┘
    ↓                ↓                ↓
Listeners        Listeners        Listeners
```

### Worker Independence

```go
// Signal A's slow handler
c.Hook(signalA, func(ctx context.Context, e *capitan.Event) {
    time.Sleep(10 * time.Second) // Very slow!
})

// Signal B's fast handler (NOT affected)
c.Hook(signalB, func(ctx context.Context, e *capitan.Event) {
    // Runs immediately, no blocking
    log.Println("Fast handler")
})

// Concurrent emission
go c.Emit(ctx, signalA, fields...) // Takes 10s
go c.Emit(ctx, signalB, fields...) // Returns immediately
```

### Ordering Guarantees

Events for the same signal are processed in order:

```go
c.Hook(orderPlaced, handler)

// These are processed in order 1, 2, 3
c.Emit(ctx, orderPlaced, orderID.Field("1"))
c.Emit(ctx, orderPlaced, orderID.Field("2"))
c.Emit(ctx, orderPlaced, orderID.Field("3"))

// Handler sees: 1 → 2 → 3 (guaranteed)
```

Different signals have no ordering guarantee:

```go
c.Emit(ctx, signalA, fields...)
c.Emit(ctx, signalB, fields...)

// No guarantee which handler runs first
```

## Thread-Safe Operations

### Concurrent Emission

Safe to emit from multiple goroutines:

```go
var wg sync.WaitGroup

for i := 0; i < 100; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()

        c.Emit(context.Background(), signal,
            idKey.Field(id),
        )
    }(i)
}

wg.Wait()
// All 100 events safely emitted
```

### Concurrent Hook Registration

Safe to register listeners concurrently:

```go
var wg sync.WaitGroup

for i := 0; i < 10; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()

        c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
            log.Printf("Listener %d: %v", id, e.Fields())
        })
    }(i)
}

wg.Wait()
// All 10 listeners registered
```

### Concurrent Observer Creation

Safe to create observers concurrently:

```go
var wg sync.WaitGroup

for i := 0; i < 5; i++ {
    wg.Add(1)
    go func(id int) {
        defer wg.Done()

        observer := c.Observe(func(ctx context.Context, e *capitan.Event) {
            log.Printf("Observer %d: %s", id, e.Signal().Name())
        })
        defer observer.Close()
    }(i)
}

wg.Wait()
```

## Shared State in Handlers

Handlers may be called concurrently across different signals. Protect shared state:

### Race Condition

```go
var counter int // Unprotected shared state

c.Hook(signal1, func(ctx context.Context, e *capitan.Event) {
    counter++ // RACE!
})

c.Hook(signal2, func(ctx context.Context, e *capitan.Event) {
    counter++ // RACE!
})
```

### Atomic Operations

```go
var counter atomic.Int64

c.Hook(signal1, func(ctx context.Context, e *capitan.Event) {
    counter.Add(1) // Safe
})

c.Hook(signal2, func(ctx context.Context, e *capitan.Event) {
    counter.Add(1) // Safe
})
```

### Mutex Protection

```go
type Counter struct {
    value int
    mu    sync.Mutex
}

func (c *Counter) Inc() {
    c.mu.Lock()
    defer c.mu.Unlock()
    c.value++
}

counter := &Counter{}

c.Hook(signal1, func(ctx context.Context, e *capitan.Event) {
    counter.Inc() // Safe
})

c.Hook(signal2, func(ctx context.Context, e *capitan.Event) {
    counter.Inc() // Safe
})
```

## Worker Queue Management

### Buffered Queues

Each worker has a buffered queue:

```go
c := capitan.New(capitan.WithBufferSize(100))

// Each signal's worker can buffer 100 events
```

### Backpressure

When a queue fills, `Emit()` blocks:

```go
c := capitan.New(capitan.WithBufferSize(5))

// Slow handler
c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    time.Sleep(1 * time.Second)
})

// Emit 10 events quickly
for i := 0; i < 10; i++ {
    c.Emit(ctx, signal, idKey.Field(i)) // Will block after 5
}

// First 5 queue immediately
// Next 5 block until queue has space
```

### Monitoring Queue Depth

Track pending events with stats:

```go
stats := c.Stats()

for signal, count := range stats.PendingEvents {
    if count > 50 {
        log.Printf("Warning: Signal %s has %d pending events", signal, count)
    }
}
```

## Context Propagation

Context flows through the entire event lifecycle:

```go
// Create context with trace ID
ctx := context.WithValue(context.Background(), "trace_id", "abc-123")

// Emit with context
c.Emit(ctx, signal, fields...)

// Handler receives same context
c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    traceID := ctx.Value("trace_id").(string) // "abc-123"

    // Use for distributed tracing
    span := tracer.StartSpan(traceID)
    defer span.Finish()
})
```

### Context Cancellation

Handlers should respect context cancellation:

```go
c.Hook(signal, func(ctx context.Context, e *capitan.Event) {
    // Check cancellation before expensive work
    select {
    case <-ctx.Done():
        log.Println("Context cancelled, aborting")
        return
    default:
    }

    // Do work
    result := expensiveOperation()

    // Check again before continuing
    select {
    case <-ctx.Done():
        log.Println("Context cancelled during operation")
        return
    default:
    }

    // Save result
    saveResult(result)
})

// Cancel context to abort in-flight operations
ctx, cancel := context.WithTimeout(context.Background(), 5*time.Second)
defer cancel()

c.Emit(ctx, signal, fields...)
```

## Graceful Shutdown

`Shutdown()` waits for all workers to drain:

```go
c := capitan.New()

// Register handlers
c.Hook(signal1, handler1)
c.Hook(signal2, handler2)

// Emit many events
for i := 0; i < 1000; i++ {
    c.Emit(ctx, signal1, fields...)
}

// Shutdown waits for all 1000 events to process
c.Shutdown()

// All events guaranteed to be processed
```

### Shutdown with Timeout

```go
func ShutdownWithTimeout(c *capitan.Capitan, timeout time.Duration) error {
    done := make(chan struct{})

    go func() {
        c.Shutdown()
        close(done)
    }()

    select {
    case <-done:
        return nil
    case <-time.After(timeout):
        return errors.New("shutdown timeout")
    }
}

// Usage
if err := ShutdownWithTimeout(c, 10*time.Second); err != nil {
    log.Printf("Warning: %v", err)
}
```

## Concurrent Patterns

### Fan-Out

Emit to multiple signals in parallel:

```go
func fanOut(c *capitan.Capitan, data Data) {
    var wg sync.WaitGroup

    // Process in parallel across signals
    wg.Add(3)

    go func() {
        defer wg.Done()
        c.Emit(ctx, signal1, fields...)
    }()

    go func() {
        defer wg.Done()
        c.Emit(ctx, signal2, fields...)
    }()

    go func() {
        defer wg.Done()
        c.Emit(ctx, signal3, fields...)
    }()

    wg.Wait()
}
```

### Scatter-Gather

Emit to workers, collect results:

```go
type ResultCollector struct {
    results []Result
    mu      sync.Mutex
}

func (rc *ResultCollector) Collect(result Result) {
    rc.mu.Lock()
    defer rc.mu.Unlock()
    rc.results = append(rc.results, result)
}

func scatterGather(c *capitan.Capitan, tasks []Task) []Result {
    collector := &ResultCollector{}

    // Hook result handler
    c.Hook(taskCompleted, func(ctx context.Context, e *capitan.Event) {
        result := resultKey.Extract(e)
        collector.Collect(result)
    })

    // Emit tasks
    for _, task := range tasks {
        c.Emit(ctx, taskStarted, taskKey.Field(task))
    }

    // Wait for results (simplified - use WaitGroup in practice)
    time.Sleep(100 * time.Millisecond)
    c.Shutdown()

    return collector.results
}
```

### Pipeline

Chain signals for sequential processing:

```go
// Stage 1: Validate
c.Hook(dataReceived, func(ctx context.Context, e *capitan.Event) {
    data := dataKey.Extract(e)

    if validate(data) {
        c.Emit(ctx, dataValidated, dataKey.Field(data))
    }
})

// Stage 2: Transform
c.Hook(dataValidated, func(ctx context.Context, e *capitan.Event) {
    data := dataKey.Extract(e)
    transformed := transform(data)

    c.Emit(ctx, dataTransformed, dataKey.Field(transformed))
})

// Stage 3: Save
c.Hook(dataTransformed, func(ctx context.Context, e *capitan.Event) {
    data := dataKey.Extract(e)
    save(data)

    c.Emit(ctx, dataSaved, dataKey.Field(data))
})
```

## Debugging Concurrency Issues

### Race Detector

Always test with race detector:

```bash
go test -race ./...
```

### Detect Deadlocks

Monitor for hung workers:

```go
func monitorWorkers(c *capitan.Capitan) {
    ticker := time.NewTicker(10 * time.Second)
    defer ticker.Stop()

    lastStats := c.Stats()

    for range ticker.C {
        currentStats := c.Stats()

        // Check if pending events are stuck
        for signal, count := range currentStats.PendingEvents {
            lastCount := lastStats.PendingEvents[signal]

            if count > 0 && count == lastCount {
                log.Printf("Warning: Signal %s has %d stuck events", signal, count)
            }
        }

        lastStats = currentStats
    }
}
```

### Trace Event Flow

Add trace IDs to track event propagation:

```go
traceKey := capitan.NewStringKey("trace_id")

// Emit with trace
traceID := generateTraceID()
c.Emit(ctx, signal1,
    traceKey.Field(traceID),
    dataKey.Field(data),
)

// Propagate trace through cascade
c.Hook(signal1, func(ctx context.Context, e *capitan.Event) {
    traceID := traceKey.Extract(e)

    log.Printf("[%s] Processing signal1", traceID)

    // Propagate to next signal
    c.Emit(ctx, signal2,
        traceKey.Field(traceID),
        resultKey.Field(result),
    )
})

c.Hook(signal2, func(ctx context.Context, e *capitan.Event) {
    traceID := traceKey.Extract(e)
    log.Printf("[%s] Processing signal2", traceID)
})
```

## Performance Considerations

### Worker Pool vs Per-Signal Workers

Capitan uses per-signal workers (not a shared pool):

**Advantages**:
- ✅ Perfect isolation between signals
- ✅ Ordered delivery per signal
- ✅ No cross-signal contention

**Tradeoffs**:
- ❌ One goroutine per signal (~2KB each)
- ❌ May over-provision for infrequent signals

For most applications, per-signal workers are better. Memory overhead is minimal.

### Optimize Buffer Sizes

```go
// Small buffers (10-50): Low memory, fast backpressure
c := capitan.New(capitan.WithBufferSize(10))

// Medium buffers (100-500): Balanced
c := capitan.New(capitan.WithBufferSize(100))

// Large buffers (1000+): Handle bursts, higher memory
c := capitan.New(capitan.WithBufferSize(1000))
```

### Minimize Observer Count

Each observer adds overhead to every `Emit()`:

```go
// ❌ Bad: 10 observers
for i := 0; i < 10; i++ {
    c.Observe(handler) // 10x overhead
}

// ✅ Good: 1 observer, dispatch internally
type Hub struct {
    handlers []capitan.EventCallback
}

func (h *Hub) Start(c *capitan.Capitan) {
    c.Observe(h.dispatch)
}

func (h *Hub) dispatch(ctx context.Context, e *capitan.Event) {
    for _, handler := range h.handlers {
        handler(ctx, e)
    }
}
```

## Best Practices

### 1. Protect Shared State

```go
// Use atomic operations or mutexes
var counter atomic.Int64
counter.Add(1)
```

### 2. Respect Context Cancellation

```go
select {
case <-ctx.Done():
    return
default:
    // Continue
}
```

### 3. Use Appropriate Buffer Sizes

```go
// Match buffer size to expected load
c := capitan.New(capitan.WithBufferSize(100))
```

### 4. Monitor Queue Depths

```go
stats := c.Stats()
for signal, count := range stats.PendingEvents {
    if count > threshold {
        alert(signal, count)
    }
}
```

### 5. Test with Race Detector

```bash
go test -race ./...
```
